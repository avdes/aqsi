Этот класс содержит в себе методы, связанные с проведением эквайринговых операций

Документация на ответ Эквайрингового ядра на запрос проведения эквайринговой операции:

Встроенный эквайринг на кассе:

Общение с эквайринговым ядром происходит в формате xml. Заботу об отправке запроса берёт на себя данная библиотека. Ответ же предстоит разобрать самостоятельно. В этом поможет описание возвращаемых параметров, приведённое ниже:

  • status - код ошибки (см. ниже)

  • header - строки заголовка чека

  • tid - идентификатор (номер) терминала

  • mid - идентификатор владельца терминала (Merchant ID)

  • seq - номер чека

  • type - тип транзакции

  • state - значение этого поля в чеке всегда active

  • tstatus - статус транзакции approved или declined

  • amount-authorized - сумма транзакции в копейках

  • amount-other- сумма кэшбека в копейках (для транзакции purchasewithcashback)

  • currency - код валюты

  • aid - идентификатор приложение EMV карты (AID)

  • appname - имя приложение EMV карты

  • pan - последние 4 цифры номера карты

  • aed - дата окончания срока действия карты (ггмм)

  • rrn - retrieval reference number

  • resp-code - код ответа сервера авторизации

  • auth-number - код авторизации.

  • datetime - дата и время проведения транзакции (yymmddHHMMSS)

  • tsi - значение EMV тэга Transaction Status Information

  • fvr - значение EMV тэга Terminal Verification Result

  • footer - строки внизу чека

  • decline-reason - значение этого тега используется только в случае

  • если транзакция отклонена. Возможные значения: unabletogoonline, offlinedeclined, systemerror, onlinedeclined, cardprocessingerror.
